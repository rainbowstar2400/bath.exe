# Bath.exe 〜いいから入れ〜 仕様書

## 1. 概要

### 1.1 コンセプト

「入りたいのに入れない」を解決するPWAアプリ。
定時にプッシュ通知で煽り、チャットで駄々をこねても最終的にお風呂に送り込む。
「今はいる！」で後押しし、「はいった！」で褒める。

### 1.2 技術スタック

| レイヤー | 技術 |
|---|---|
| フロント | PWA（HTML/JS）+ Service Worker |
| バックエンド | Vercel サーバーレス関数 |
| プッシュ通知 | Web Push API + `web-push`（VAPID認証） |
| AI | Claude API（claude-sonnet-4-20250514） |
| DB | Supabase |
| 定時実行 | Supabase pg_cron + pg_net |
| 認証 | Supabase 匿名認証 |

## 2. 機能仕様

### 2.1 プッシュ通知（段階的煽り）

設定時刻を起点に3回、段階的にプッシュ通知を送信する。
デフォルトの通知開始時刻は **23:00**。

| 段階 | タイミング | 強度 | タイトル例 | 本文例 |
|---|---|---|---|---|
| 1回目 | 設定時刻 +0分 | やさしめ |
| 2回目 | 設定時刻 +15分 | 煽り |
| 3回目 | 設定時刻 +30分 | 圧 |

- **通知文は Claude API で毎回動的生成する**（段階と強度を指定し、毎日異なる煽り文を生成）
- 通知にはバイブレーションパターンを設定する（`vibrate` オプション）
- 段階が進むほどバイブレーションを長くする
- 通知タップでアプリのチャット画面を開く
- 送信前に「入った！」フラグをチェックし、済みならスキップする

### 2.2 チャット（駄々こね機能）

Claude APIを使い、ユーザーの言い訳に対応するチャット機能。

**システムプロンプト方針：**
- 共感はするけど絶対に折れない
- 図星をついてくるトーン
- どんな言い訳にも切り返す
- 最終的には必ず「入るよ！」に着地させる

**セッション管理：**
- 会話履歴は `chat_sessions` テーブルに保存
- 毎日 12:00（正午）にセッションをリセット（新セッション開始）
- Claude API にはセッション内の全会話履歴を送信し、文脈を維持する

**メッセージ上限：**
- 1セッションあたり **10往復** まで
- 上限に達したらチャット入力を無効化し、「もう言い訳は聞かない。入れ。」のようなメッセージを表示
- 画面上にも残り回数を表示する（例：「残り 7/10」）

### 2.3 入浴ボタン（2段階）

メイン画面下部に大きく配置する入浴アクションボタン。2段階の操作フローで入浴を促す。

**フェーズ1：「今はいる！」**
- タップすると、チャットにユーザー吹き出し「今はいる！」を表示
- AIが後押しメッセージを返す（例：「いいぞいいぞ！いってらっしゃい！」）
- ボタンが「はいった！」に変化（ブルー → コーラルに色変化）

**フェーズ2：「はいった！」**
- お風呂から出た後にタップ
- チャットにユーザー吹き出し「はいった！」を表示
- `POST /api/bath/done` で記録 + Claude API で褒めメッセージを生成
- ボタンが「おつかれ！」に変化（グリーンに色変化、操作不可）
- 未送信の残り通知をキャンセルする

**褒め方のルール：**
- 設定時刻より前に押した → 激褒め（例：「え、早くない！？天才！？」）
- 設定時刻〜+30分以内に押した → 普通に褒める（例：「えらい！おつかれ！」）
- +30分を過ぎて押した → 「入っただけ偉い！」のトーン

**チャット表示の区別：**
- ボタンアクションによる送信/返答はブルー系の色で表示
- 通常の言い訳チャットはピーチ（ユーザー）/ 白（AI）で表示
- 各メッセージに送信時刻を表示

## 3. 画面仕様

### 3.1 メイン画面

```
┌─────────────────────────┐
│  📊              ⚙️     │
│                         │
│  ┌───────────────────┐  │
│  │                   │  │
│  │  チャットエリア      │  │
│  │  （中央にBath.exe   │  │
│  │   ウォーターマーク） │  │
│  │                   │  │
│  └───────────────────┘  │
│                         │
│  [ 言い訳を入力… ][送信] │
│              残り 7/10  │
│  [     今はいる！      ] │
└─────────────────────────┘
```

**レイアウト：**
- 上部ナビ：📊（きろく、左）、⚙️（設定、右）のアイコンのみ
- チャットエリア：flex:1で画面中央を占有、背景にBath.exeウォーターマーク
- 下部アクションエリア：入力欄 → 残り回数 → 入浴ボタンの順に配置

**要素：**
- チャットエリア：会話履歴をスクロール表示（送信時刻付き）
- メッセージ入力欄＋送信ボタン（上限到達で無効化）
- 残り回数表示（10往復中の残数）
- 入浴ボタン：2段階操作（2.3参照）、画面最下部に横幅いっぱいで配置

```
┌─────────────────────────┐
```

**要素：**
- 通知時刻：タイムピッカーで設定（変更時にpg_cronジョブも更新）
- 通知オンオフ：トグルスイッチ
- 通知許可ボタン：ブラウザの通知許可が未取得の場合のみ表示

## 4. API 仕様

### 4.1 `POST /api/subscriptions/register`

サブスクリプション情報を登録・更新する。

**リクエスト：**
```json
{
  "endpoint": "https://fcm.googleapis.com/...",
  "keys": {
    "auth": "...",
    "p256dh": "..."
  }
}
```

**処理：**
1. Supabase 匿名認証のトークンからユーザーIDを取得
2. `subscriptions` テーブルに upsert

### 4.2 `POST /api/push/send`

pg_cron から叩かれ、対象ユーザーにプッシュ通知を送信する。

**リクエスト：**
```json
{
  "stage": 1
}
```

**処理：**
1. `enabled = true` かつ現在時刻が通知対象のユーザーを `subscriptions` から取得
2. 当日の「入った！」フラグが立っていたらスキップ
3. 段階（stage）に応じた通知文を Claude API で動的生成
4. `web-push` ライブラリで送信
5. 送信失敗で 410 が返った場合、該当サブスクリプションを DB から削除（期限切れ処理）

**認証：** pg_cron からの呼び出しのため、シークレットキーで認証する。

### 4.3 `POST /api/chat`

チャットメッセージを処理する。

**リクエスト：**
```json
{
  "message": "あと5分だけ..."
}
```

**処理：**
1. ユーザーIDから当日のセッションを取得（なければ新規作成）
2. 会話履歴にユーザーメッセージを追加
3. システムプロンプト＋会話履歴を Claude API に送信
4. レスポンスを会話履歴に追加して保存
5. レスポンスを返却

**レスポンス：**
```json
{
  "reply": "5分後って言って5分で済んだことある？ほら、立って。"
}
```

### 4.4 `POST /api/bath/done`

「入った！」を処理する。

**処理：**
1. 当日の残りpg_cronジョブをキャンセル（または送信済みフラグを立てる）
2. 現在時刻と設定時刻を比較して褒め度合いを判定
3. Claude API で褒めメッセージを生成
4. レスポンスを返却

**レスポンス：**
```json
{
  "reply": "え、まだ22時50分！？早すぎ！天才！偉すぎる！！！",
  "praise_level": "excellent"
}
```

### 4.5 `GET /api/stats`

統計データを取得する。

**レスポンス：**
```json
{
  "streak": 5,
  "weekly": {
    "rate": 0.71,
    "days": [
      { "date": "2026-03-23", "done_at": "2026-03-23T22:48:00+09:00", "praise_level": "excellent" },
      { "date": "2026-03-24", "done_at": "2026-03-24T23:12:00+09:00", "praise_level": "good" },
      { "date": "2026-03-26", "done_at": null, "praise_level": null }
    ]
  }
}
```

**処理：**
1. `bath_logs` から直近7日分のデータを取得
2. 連続記録（streak）を算出
3. 週間入浴率を算出

### 4.6 `POST /api/settings/update`

通知設定を更新する。

**リクエスト：**
```json
{
  "notify_time": "22:30",
  "enabled": true
}
```

**処理：**
1. `subscriptions` テーブルの該当レコードを更新
2. pg_cron のジョブを `cron.unschedule` → `cron.schedule` で再登録

## 5. DB スキーマ

### 5.1 subscriptions

| カラム | 型 | 制約 | 説明 |
|---|---|---|---|
| id | uuid | PK, default gen_random_uuid() | |
| user_id | uuid | NOT NULL, UNIQUE | Supabase auth のユーザーID |
| endpoint | text | NOT NULL | プッシュエンドポイントURL |
| auth | text | NOT NULL | VAPID auth キー |
| p256dh | text | NOT NULL | VAPID p256dh キー |
| notify_time | time | NOT NULL, default '23:00' | 通知開始時刻 |
| enabled | boolean | NOT NULL, default true | 通知オンオフ |
| created_at | timestamptz | default now() | 作成日時 |
| updated_at | timestamptz | default now() | 更新日時 |

### 5.2 chat_sessions

| カラム | 型 | 制約 | 説明 |
|---|---|---|---|
| id | uuid | PK, default gen_random_uuid() | |
| user_id | uuid | NOT NULL | Supabase auth のユーザーID |
| messages | jsonb | NOT NULL, default '[]' | 会話履歴（Claude API形式） |
| session_date | date | NOT NULL | セッションの日付 |
| created_at | timestamptz | default now() | 作成日時 |

**ユニーク制約：** `(user_id, session_date)`

**セッション日付の判定：**
- 正午（12:00 JST）を日付の境界とする
- 例：3/27 12:00 〜 3/28 11:59 → session_date = '2025-03-27'

### 5.3 bath_logs

「入った！」の記録。統計画面で使用する。

| カラム | 型 | 制約 | 説明 |
|---|---|---|---|
| id | uuid | PK, default gen_random_uuid() | |
| user_id | uuid | NOT NULL | Supabase auth のユーザーID |
| done_at | timestamptz | NOT NULL, default now() | お風呂完了時刻 |
| notify_time | time | NOT NULL | その日の設定時刻（比較用） |
| praise_level | text | NOT NULL | excellent / good / late |

## 6. プッシュ通知フロー

### 6.1 初回登録

```
ユーザー → アプリ起動
         → 通知許可ダイアログ
         → 許可
         → Service Worker がサブスクリプション取得
         → POST /api/subscriptions/register
         → Supabase に保存
```

### 6.2 定時通知

```
pg_cron (23:00 JST)
  → pg_net で POST /api/push/send { stage: 1 }
  → Vercel 関数が Supabase から対象ユーザーを取得
  → web-push で通知送信
  → スマホに通知表示

pg_cron (23:15 JST)
  → POST /api/push/send { stage: 2 }
  →（同上）

pg_cron (23:30 JST)
  → POST /api/push/send { stage: 3 }
  →（同上）
```

### 6.3 「入った！」による通知キャンセル

```
ユーザー → 「入った！」ボタン
         → POST /api/bath/done
         → 当日の残り通知をスキップするフラグを立てる
         → /api/push/send は送信前にフラグをチェックし、スキップ
         → 褒めメッセージを返却
```
